# BoardGraph

An interactive network visualisation of biotech and pharma board interlocks — who
sits on whose boards, and how companies, universities, VC firms, and regulators
are connected through shared directors.

---

## Folder structure

```
boardgraph/
├── data/
│   ├── companies_market_cap.csv   ← source list of companies, ranked by market cap
│   ├── people.json                ← master list of people & affiliations (edit freely)
│   ├── orgs.json                  ← organisation registry (edit freely)
│   ├── pipeline_progress.json     ← auto-generated; tracks which companies are done
│   └── raw/                       ← auto-generated scrape outputs; never edit manually
│       ├── sec/                   ← DEF 14A proxy filing data per ticker
│       ├── wikidata/              ← Wikidata board member results per org
│       └── wikipedia/             ← scraped Wikipedia pages
├── scripts/
│   ├── requirements.txt
│   ├── pipeline.py                ← main entry point; works through the CSV interactively
│   ├── build_data.py              ← compiles people.json + orgs.json → site/data.js
│   ├── scrape_sec.py              ← fetches DEF 14A filings from SEC EDGAR
│   ├── scrape_wikidata.py         ← queries Wikidata SPARQL for board members
│   └── scrape_wikipedia.py        ← scrapes Wikipedia for affiliation data
└── site/
    ├── index.html                 ← the website (never edit data here directly)
    └── data.js                    ← GENERATED by build_data.py; do not edit directly
```

## Quick start

```bash
# 1. Install dependencies
pip install -r scripts/requirements.txt

# 2. Add your Anthropic API key to a .env file in the project root
#    (the pipeline uses Claude to extract board data from SEC filings)
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env

# 3. Run the pipeline — processes the next company in the CSV interactively
python scripts/pipeline.py

# 4. Rebuild the website data file when you're done adding people
python scripts/build_data.py

# 5. Serve the site locally
cd site && python -m http.server 8080
# → open http://localhost:8080
```

> **Why a local server?** `index.html` loads `data.js` via a `<script>` tag.
> Browsers block this on `file://` URLs. Python's built-in server works fine;
> VS Code Live Server or any static host also works.

> **Windows PowerShell:** Use a `.env` file as shown above rather than `export`.
> The pipeline loads it automatically via `python-dotenv`.

## The pipeline

`pipeline.py` is the primary way to populate the dataset. It works through
`data/companies_market_cap.csv` in rank order (largest market cap first),
finding board members for each company and asking you to approve them before
writing anything to disk.

### Basic usage

```powershell
# Process the next unprocessed company in the CSV
python scripts/pipeline.py

# Jump to a specific company by name
python scripts/pipeline.py --company "Merck"

# Process companies in sequence, pausing between each
python scripts/pipeline.py --loop

# Run fully automated — approves all candidates, no TUI
python scripts/pipeline.py --loop --auto

# Check progress
python scripts/pipeline.py --status

# Mark a company as unprocessed so it gets re-run
python scripts/pipeline.py --reset "Eli Lilly"
```

### How it finds board members

For each company, both Wikidata and SEC are queried and their results merged:

| Source | How it works | Notes |
|--------|-------------|-------|
| **Wikidata** | SPARQL query for `P3320` (board member) triples | Clean structured data; may lag on recent appointments |
| **SEC DEF 14A → Claude** | Downloads the proxy filing, locates the director section, passes text to Claude API, gets structured JSON back | Catches recent appointments Wikidata misses |
| **Wikipedia → Claude** | Fetches the Wikipedia article, passes to Claude | Fallback for companies with no SEC filing |

Wikidata results are filtered to remove former members (those with a
`director_until` date set). SEC results are merged in to catch anyone Wikidata
is missing. The two sources complement each other well — Wikidata is cleaner,
SEC is more current.

Duplicate entries caused by middle initials (`Jennifer Doudna` vs
`Jennifer A. Doudna`) are automatically merged, keeping the longer name and
combining affiliations from both sources.

### The approval UI

After finding candidates the pipeline shows them one at a time:

```
════════════════════════════════════════════════════════════
  #1  Eli Lilly  (LLY)  $789.7B  United States
════════════════════════════════════════════════════════════

  1/12  David A. Ricks
  Chair & CEO  ·  since 2017
  Other: Adobe Inc.
  [y/n/e/a/q]
```

| Key | Action |
|-----|--------|
| `y` | Approve and write to people.json |
| `n` | Skip this person |
| `e` | Edit name, role, or affiliations before approving |
| `a` | Approve this person and all remaining candidates at once |
| `q` | Stop reviewing this company (keeps what you've approved so far) |

### Affiliation expansion

After approvals, any organisations mentioned in directors' affiliations that
aren't already in `orgs.json` are surfaced:

```
  8 new organisations found in affiliations:
    · Stanford University
    · Howard Hughes Medical Institute
    · The Procter & Gamble Company
    …
  Add all as new orgs? [y/n/s=select]
```

Choosing `s` lets you go through them one at a time. This is how the dataset
snowballs — processing J&J surfaces Intellia, Tempus AI, JPMorganChase, Duke
University, and so on, which can then be processed in future pipeline runs.

Added orgs get a guessed type and a Wikipedia slug so they can be enriched
later by `scrape_wikidata.py`.

## Manual data editing

### people.json

Each person has these fields:

```json
{
  "id":             "Katherine Baicker",
  "initials":       "KB",
  "role":           "Independent Director",
  "category":       "academic",
  "orgs":           ["Eli Lilly", "Harvard University"],
  "notes":          "",
  "last_updated":   "2026-04-01",
  "sources":        ["https://www.sec.gov/..."],
  "director_since": "2019",
  "director_until": ""
}
```

`category` must be one of: `academic` · `executive` · `vc` · `regulatory`

`orgs` must contain IDs that exist in `orgs.json`. Run
`python scripts/build_data.py --validate` to catch typos.

### orgs.json

Each organisation:

```json
{
  "id":        "Eli Lilly",
  "type":      "pharma",
  "ticker":    "LLY",
  "cik":       "0000059478",
  "wikipedia": "Eli_Lilly_and_Company",
  "notes":     ""
}
```

`type` must be one of: `biotech` · `pharma` · `tech` · `academia` · `vc` · `other`

`cik` is the SEC EDGAR company identifier — the pipeline looks this up
automatically from EDGAR's company ticker file when a new org is added.
You can also find it manually at
[efts.sec.gov](https://efts.sec.gov/LATEST/search-index?q=%22eli+lilly%22&forms=DEF+14A).

`wikipedia` is the page slug (the part after `en.wikipedia.org/wiki/`) — used
by `scrape_wikidata.py` to resolve the Wikidata QID.

### Recording a board change

1. Find the person in `people.json`
2. Add or remove the org from their `"orgs"` array
3. Update `"last_updated"` to today's date
4. Set `"director_until"` if they left, or `"director_since"` if newly joined
5. Add a source URL (SEC 8-K press release, company IR page, etc.)
6. Run `python scripts/build_data.py`

## build_data.py

Compiles `people.json` and `orgs.json` into `site/data.js` for the website.

```bash
python scripts/build_data.py                  # rebuild site/data.js
python scripts/build_data.py --dedup          # merge duplicate people (middle-initial mismatches)
python scripts/build_data.py --merge-raw      # import raw scrape outputs as draft entries
python scripts/build_data.py --auto-approve   # approve drafts that pass quality filters
python scripts/build_data.py --validate       # check for broken org references
python scripts/build_data.py --stats          # show people counts by source and status
```

### Deduplication

Running `--dedup` merges entries where the same person was added twice under
slightly different names (e.g. `Jennifer Doudna` and `Jennifer A. Doudna`).
It keeps the longer name, unions the org lists, and fills in any missing fields
from the richer entry. Run this after the first few pipeline sessions to clean
up any duplicates that got approved before the automatic merge was in place.

### The review / auto-approve workflow

`pipeline.py` writes people as pre-approved — they appear in the website
immediately. The older scraper scripts (`scrape_sec.py`, `scrape_wikidata.py`)
write to `data/raw/` and require a separate merge + approval step:

```bash
python scripts/build_data.py --merge-raw      # import as review drafts
python scripts/build_data.py --auto-approve   # approve those that pass quality checks
python scripts/build_data.py                  # rebuild site
```

The auto-approve quality gate by source:

| Source | Auto-approved if… |
|--------|--------------------|
| Wikidata | Name passes quality filter — structured data, trusted unconditionally |
| SEC DEF 14A | Name passes quality filter **and** `director_since` year is present |
| Wikipedia | Always kept for manual review — too noisy to auto-approve |

## Individual scrapers

These can be run independently to refresh data without the interactive pipeline.

### scrape_wikidata.py

```bash
python scripts/scrape_wikidata.py                   # all orgs with wikipedia slugs
python scripts/scrape_wikidata.py --org "Moderna"   # single org
python scripts/scrape_wikidata.py --type biotech    # one category
python scripts/scrape_wikidata.py --dry-run         # resolve QIDs only, no queries
```

Results go to `data/raw/wikidata/`. Run
`python scripts/build_data.py --merge-raw --auto-approve` afterwards.

### scrape_sec.py

```bash
python scripts/scrape_sec.py                    # all public companies in orgs.json
python scripts/scrape_sec.py --ticker MRNA      # single company
python scripts/scrape_sec.py --dry-run          # print doc URLs without downloading
```

Results go to `data/raw/sec/`. SEC EDGAR allows ~10 requests/second; the script
sleeps 150ms between requests. Note that the pipeline's Claude-based extraction
is generally more accurate than this scraper for complex iXBRL filings.

### scrape_wikipedia.py

```bash
python scripts/scrape_wikipedia.py --orgs     # scrape each org's Wikipedia page
python scripts/scrape_wikipedia.py --people   # look up each person's Wikipedia page
```

Results go to `data/raw/wikipedia/`. Useful for private companies and academic
institutions without SEC filings or Wikidata entries.

## Keeping data current

DEF 14A proxy filings are submitted annually, typically 3–4 months after fiscal
year end. Board changes between filings are announced via SEC 8-K filings within
4 business days.

| Cadence | Action |
|---------|--------|
| Weekly / as time allows | `python scripts/pipeline.py --loop` — continue down the CSV |
| Quarterly | Re-run `scrape_wikidata.py` for existing orgs to catch board changes |
| As needed | Manually update `people.json` for changes you see in the news |
| Annually | Re-run `scrape_sec.py` to refresh all DEF 14A data |

## Website

The site is a self-contained static page at `site/index.html`. No build step or
server-side code required — it reads `site/data.js` generated by `build_data.py`.

**Graph interactions:**
- Nodes are people, sized by connection count, coloured by role category
- Edges connect two people who share at least one org
- Left sidebar — click any org to highlight its connected people without
  reshuffling the graph layout; multiple orgs stack
- Hover a node — tooltip shows all affiliations colour-coded by org type
- Click a node — detail panel slides in with role, affiliations, and co-board
  connections sorted by number of shared orgs
- Search bar — filters by name, role, or any org name
- Scroll to zoom, drag to pan

**Role colours:**
- Green — Academic / Scientist
- Blue — Industry Executive
- Orange — VC / Investor
- Purple — Regulatory / Policy
